Gerando uma chave SSH e adicionando em sua conta GITHUB no linux

Aprenda a gerar uma chave SSH e autenticar seu acesso seja no GITHUB ou em seu servidor remoto

Publicado por Lucas Lagrimante em 23 Julho de 2017

Sobre SSH

Usando o protocolo SSH, você pode se conectar e autenticar servidores e serviços remotos. Com as chaves SSH, você pode se conectar ao GitHub sem fornecer seu nome de usuário ou senha em cada visita. Quando você configura o SSH, você gerará uma chave SSH e vai adiciona-la ao ssh-agent e, em seguida, adicionar a chave à sua conta do GitHub. Adicionar a chave SSH ao ssh-agent garante que sua chave SSH tenha uma camada extra de segurança através do uso de uma frase secreta.

Gerando uma nova chave SSH

1 - Abra o terminal

2 - Copie o texto abaixo e substitua o email por seu email

ssh-keygen -t rsa -b 4096 -C "seu_email@exemplo.com"

Isso cria uma nova chave com seu email de rótulo

Gerando par de chaves rsa pública / privada.

3 - Quando você for solicitado a "Inserir um arquivo no qual salvar a chave", pressione Enter. Isso aceita a localização padrão do arquivo.

Enter a file in which to save the key (/home/você/.ssh/id_rsa): [Aperte Enter]

4 - No terminal, digite uma frase secreta segura.

Enter passphrase (empty for no passphrase): [Digite a chave secreta] Enter same passphrase again: [Digite a chave secreta]

Antes de adicionar uma nova chave SSH ao ssh-agent para gerenciar suas chaves, você deve verificar as chaves SSH existentes e gerar uma nova chave SSH.

1 - Inicie o ssh-agent em segundo plano.

$ eval "$(ssh-agent -s)" Agent pid 59566

2 - Adicione sua chave privada SSH ao ssh-agent. Se você criou sua chave com um nome diferente, ou se você estiver adicionando uma chave existente que tenha um nome diferente, substitua id_rsa no comando com o nome do arquivo de chave particular.

$ ssh-add ~/.ssh/id_rsa
