Instalando o Redmine no Ubunto passo a passo

Aprenda a instalar e configurar um dos maiores e mais conhecidos controladores de tarefas

Publicado por Lucas Lagrimante em 22 Julho de 2017

Introdução

Este tutorial orienta você passo a passo através da instalação do Redmine em uma instalação limpa do Ubunto 16.04. Isso se destina a ser um método completo para instalar e executar o Redmine. Não lidaremos com outras aplicações instaladas ou configuradas. Estaremos fazendo isso a partir do prompt da linha de comando para manter as coisas tão claras e limpas quanto possível. Recomendo que você instale as atualizações do Ubuntu antes de iniciar este processo. Há quase sempre alguma esperando para ser aplicada depois que o Ubuntu for configurado pela primeira vez.

Pré-Requisitos Apache, mod-passenger, e MySQL

1 - A instalação do apache é simples, apenas siga os passos padrões

$ sudo apt-get install apache2 libapache2-mod-passenger

2 - Instalar o mysql leva apenas um pouco mais de tempo, então os detalhes serão explicados

$ sudo apt-get install mysql-server mysql-client

O processo de instalação do mysql irá pedir-lhe uma senha para o acesso "root" para o servidor do banco de dados e, em seguida, pedir-lhe para confirmar a senha em uma tela de acompanhamento. Isso define a senha de administração do banco de dados.

Certifique-se de que a gema do pacote esteja instalada

$ sudo gem update
$ sudo gem install bundler

Configurando o Apache

1 - Você precisa modificar dois arquivos para apache. O primeiro é /etc/apache2/mods-available/passenger.conf que precisa do texto PassengerDefaultUser www-data como mostrado abaixo. Não tente substituir o conteúdo completo do arquivo.

<IfModule mod_passenger.c>
  PassengerDefaultUser www-data
  PassengerRoot /usr
  PassengerRuby /usr/bin/ruby
</IfModule>

2 - Agora, crie um link simbólico para conectar o Redmine no espaço do documento da Web:

$ sudo ln -s /usr/share/redmine/public /var/www/html/redmine

3 - E modifique /etc/apache2/sites-available/000-default.conf (antes do apache 2.4 /etc/apache2/sites-available/default) para inserir o seguinte com as outras seções do <Directory> para que o apache saiba seguir o link simbólico:

<Directory /var/www/html/redmine>
    RailsBaseURI /redmine
    PassengerResolveSymlinksInDocumentRoot on
 </Directory>

4 - Crie e configure a propriedade de um arquivo Gemfile.lock para que o usuário do www-data do apache possa acessá-lo:

$ sudo touch /usr/share/redmine/Gemfile.lock
$ sudo chown www-data:www-data /usr/share/redmine/Gemfile.lock

5 - Restarte o apache:

$ sudo service apache2 restart

6 - Agora você já pode acessar o endereço do Redmine e usa-lo!

$ firefox http://127.0.0.1/redmine
